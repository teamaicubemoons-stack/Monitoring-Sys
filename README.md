# Team Task Management & Employee Monitoring System

This is a complete enterprise-level Task Management and Employee Monitoring web application. The platform allows organizations to manage teams, assign tasks, and monitor employee activities in real-time. 

Instead of a traditional SQL database, this system is uniquely configured to use **Google Sheets** as its primary database.

## 🛠 Tech Stack
- **Frontend**: React.js (Vite), Tailwind CSS, Framer Motion, Socket.IO Client, Axios
- **Backend**: Python, FastAPI, WebSockets, JWT Authentication, Passlib (Bcrypt)
- **Database**: Google Sheets (via `gspread`)

---

## 🚀 Setup Instructions

### 1. Google Sheets Database Setup (Crucial)
Because we are using Google Sheets as our database, you **must** provide Google Service Account credentials.

1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
2. Create a New Project.
3. In the search bar, look for **Google Drive API** and **Google Sheets API**, and **Enable** both of them.
4. Go to **APIs & Services > Credentials**.
5. Click **Create Credentials > Service Account**.
6. Fill in the details, click "Create and Continue", and grant it an "Editor" role.
7. Click on your newly created Service Account, go to the **Keys** tab.
8. Click **Add Key > Create New Key > JSON**. This will download a `.json` file to your computer.
9. Rename this downloaded file to `credentials.json`.
10. **Move `credentials.json`** inside the `backend` folder of this project.

*(Optional)*: Open your `backend/.env` file and set a custom name for your sheet:
```env
SPREADSHEET_NAME=MonitoringSystemDB
SECRET_KEY=your_super_secret_jwt_key
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=480
```

---

### 2. Backend Setup
The backend runs on Python and FastAPI.

Open a new terminal and navigate to the project folder:
```bash
# Navigate to backend directory
cd "Monitoring system\backend"

# Create a virtual environment (Recommended)
python -m venv venv
venv\Scripts\activate   # For Windows

# Install all backend dependencies
pip install -r requirements.txt

# Seed the Google Sheet Database with initial dummy data (Admin & Employee)
python init_db.py

# Start the FastAPI Server
uvicorn main:app --reload
```
**Backend API will be running at:** `http://localhost:8000`

---

### 3. Frontend Setup
The frontend runs on React & Vite.

Open a **second** terminal window and navigate to the frontend folder:
```bash
# Navigate to frontend directory
cd "Monitoring system\frontend"

# Install frontend dependencies
npm install

# Start the React Development Server
npm run dev
```
**Frontend Web App will be running at:** `http://localhost:5173` (Check the terminal for the exact Vite localhost URL).

---

### 4. How to Use the Application

1. Open your browser and go to the frontend URL (e.g., `http://localhost:5173`).
2. You will be greeted by the Login screen.
3. Log in using the test credentials generated by `init_db.py`:
   - **Admin Login**: `admin@company.com` | Password: `admin123`
   - **Employee Login**: `employee@company.com` | Password: `password123`
4. Once logged in, the `ActivityTracker` component activates in the background. It will track your mouse, keyboard, and tab visibility. 
5. If you do not interact with the page for 5 minutes, your status will automatically be updated to **Idle** via WebSockets!
6. Go check your Google Drive! A new Google Sheet (named `MonitoringSystemDB`) will be created automatically, containing tabs for `Users`, `Tasks`, and `ActivityLogs`.

## ⚠️ Important Note regarding Google Sheets Limits
Because we are using WebSockets to monitor *real-time* activity (mouse movements, tab switches), the frontend will attempt to send frequent updates. 
**Google Sheets has a strict rate limit of 60 writes per minute per user.**
If you have multiple employees active at the same time, you may hit the API quota limit rapidly. For a true production environment, consider adding a caching layer (like Redis) or writing logs in batches every 5 minutes rather than instantly.
